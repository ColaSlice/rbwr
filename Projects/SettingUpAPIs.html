Creating An API

There are a couple of variables to consider when creating an API. You need to know what the type of API it needs to be, what it will do, how it will do it, how to deploy it and where to deploy it. Of cause, there is more to this, but this is just the basics.

What my API should be is a rest API, it will return basic system usage of the system when called, it will be made with Python Flask with the psutil package, it will be deployed with Gunicorn and it will be deployd on my VPS. I want the API to only be accessible locally on the VPS. Meaning if you are trying to call it from outside the VPS, nothing will happen. Only processes on the VPS can call it.

To run the API, simply do gunicorn -b 127.0.0.1:9000 app:app. The first command gunicorn runs the program gunicorn. The argument -b tells gunicorn to not expose the IP and port to the outside of the network. The first part of the argument 127.0.0.1:9000 is the IP, the : is meant to tell where the IP ends and the port start. The second part 9000 is the port number for the API to use. The last agument app:app is to tell wich Python scripts to use. The script is called app, and is of type app.
